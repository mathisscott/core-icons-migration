Clarity Icons Migration:

  • Import the Clarity Core ClarityIcons service from @clr/core/icon. This service will be used to add icon shapes to the library as needed.
  • Import any icon shapes from @clr/core/icon. An IDE like VS Code might be helpful in auto-completing the shape imports.
  • Add the imported icons to the ClarityIcons service using the ClarityIcons.addIcons() method. Icons can be added at the module level or component level. Adding an icon more than once to ClarityIcons will not have a negative effect.
  • Import the registration file at @clr/core/icon/register. This file adds the cds-icon component to the browser. It's best if the file is imported at the app module level but there is no harm in importing it at the component level. Importing the registration file more than once has no negative effect.
  • To use web components in an Angular application, the CUSTOM_ELEMENTS_SCHEMA needs to be defined on the application module. Import CUSTOM_ELEMENTS_SCHEMA from @angular/core and make sure to add CUSTOM_ELEMENTS_SCHEMA to the schemas array property of your application's @NgModule decorator.
